## Prerequisites

- Docker (recommended) or Python 3.10+
- An AI coding agent: Claude Code, Cursor, or any MCP client

## 1. Start the Clean server

<Tabs>
  <Tab title="Docker (recommended)">
    ```bash
    git clone https://github.com/useclean/Clean.git
    cd Clean

    cp .env.example .env
    # Edit .env — set CLEAN_API_KEY and CLEAN_LICENSE_KEY

    docker compose up -d
    ```

    The API is now running at `http://localhost:8000`.
  </Tab>
  <Tab title="Local Python">
    ```bash
    git clone https://github.com/useclean/Clean.git
    cd Clean

    make setup   # Creates venv, installs deps, generates .env
    make dev     # Starts server on localhost:8000 with hot reload
    ```
  </Tab>
</Tabs>

## 2. Index a repository

```bash
curl -X POST "http://localhost:8000/repos/index?repo=owner/repo" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Clean will clone the repo, parse all Python/JavaScript/TypeScript files, embed them, and build the call graph. You can check progress:

```bash
curl "http://localhost:8000/repos/owner/repo/status" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 3. Search your code

```bash
curl "http://localhost:8000/search?q=authentication+logic&repo=owner/repo" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Results come back in TOON format — a compact tabular format that saves 30-40% tokens vs JSON.

## 4. Connect your AI agent

Add Clean as an MCP server so your agent can search directly.

<Tabs>
  <Tab title="Claude Code">
    Add to `~/.claude/settings.json`:

    ```json
    {
      "mcpServers": {
        "clean": {
          "url": "http://localhost:8000/mcp/sse",
          "headers": {
            "Authorization": "Bearer YOUR_API_KEY"
          }
        }
      }
    }
    ```
  </Tab>
  <Tab title="Cursor">
    Add to `.cursor/mcp.json`:

    ```json
    {
      "mcpServers": {
        "clean": {
          "url": "http://localhost:8000/mcp/sse",
          "headers": {
            "Authorization": "Bearer YOUR_API_KEY"
          }
        }
      }
    }
    ```
  </Tab>
</Tabs>

Restart your agent. It now has access to `search_code`, `index_repo`, and `list_repos` tools.

## 5. Try it out

Ask your agent: *"Search for the payment processing logic"*

The agent will call `search_code` through MCP and receive semantically relevant results with call graph context — callers, callees, and related functions in the same file.

<Tip>
  Clean auto-detects stale indexes. If your repo has changed since the last index, it will re-index automatically on the next search.
</Tip>
