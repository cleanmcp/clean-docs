## How agents connect

Clean exposes an MCP server that AI agents connect to over **SSE** (Server-Sent Events) or **stdio**. Once connected, your agent gets access to these tools:

| Tool | What it does |
|------|-------------|
| `search_code` | Semantic code search with call graph context |
| `index_repo` | Index a GitHub repository |
| `list_repos` | List all indexed repositories |
| `get_token_savings` | Show token savings statistics |

## Claude Code

Add to your `~/.claude/settings.json`:

```json
{
  "mcpServers": {
    "clean": {
      "url": "http://localhost:8000/mcp/sse",
      "headers": {
        "Authorization": "Bearer YOUR_API_KEY"
      }
    }
  }
}
```

Restart Claude Code. You can verify by asking: *"List all indexed repos"* — it should call the `list_repos` tool.

## Stdio mode (direct integration)

For agents that support MCP over stdio (no HTTP server needed):

```bash
pip install clean
```

Then configure your agent to run the `clean` command:

```json
{
  "mcpServers": {
    "clean": {
      "command": "clean",
      "args": ["--repo", "owner/repo"]
    }
  }
}
```

The `--repo` flag pins a default repository so the agent doesn't need to specify it on every search.

## How search works for agents

When your agent calls `search_code`, it sends a natural language query and gets back:

1. **Matching functions** with full source code, file path, and line numbers
2. **Call graph context** — callers, callees, and same-file functions
3. **Similarity scores** showing how closely each result matches the query

The results are formatted for minimal token usage. Your agent can answer directly from the returned code without needing to read additional files.

<Tip>
  Clean auto-detects the repository when your agent passes its current working directory via the `cwd` parameter. If only one repo is indexed, Clean auto-selects it — no `repo` parameter needed.
</Tip>

## Verifying the connection

After configuring your agent, try these prompts:

1. **"List indexed repos"** — calls `list_repos`, confirms the connection works
2. **"Search for authentication logic"** — calls `search_code`, confirms search works
3. **"Show token savings"** — calls `get_token_savings`, shows cumulative savings

If the agent can't find the tools, check:
- The Clean server is running (`curl http://localhost:8000/`)
- Your API key is correct (`Authorization: Bearer YOUR_KEY`)
- The MCP SSE endpoint is reachable (`http://localhost:8000/mcp/sse`)
