## Overview

Clean can integrate with GitHub as a GitHub App. When installed on a repository, it automatically:

1. **Indexes all repos** on installation — clones and indexes every selected repository
2. **Re-indexes on push** — pulls latest changes and runs incremental re-indexing when code is pushed
3. **Cleans up on uninstall** — removes cloned repos and index data when the app is uninstalled

## Setting up the GitHub App

### 1. Create the App

Go to **GitHub Settings > Developer settings > GitHub Apps > New GitHub App** and configure:

| Setting | Value |
|---------|-------|
| App name | `Clean` (or your preferred name) |
| Homepage URL | `https://useclean.dev` |
| Webhook URL | `https://your-server.com/github/webhook` |
| Webhook secret | Generate a random secret |
| Repository permissions | **Contents**: Read-only |
| Events | **Push** |

### 2. Generate a private key

After creating the app, generate a private key (`.pem` file) and save it securely.

### 3. Configure Clean

Set the following environment variables:

```bash
GITHUB_APP_ID=123456
GITHUB_PRIVATE_KEY_PATH=/path/to/your-app.pem
GITHUB_WEBHOOK_SECRET=your-webhook-secret
```

In Docker, mount the private key file:

```yaml
services:
  clean:
    volumes:
      - clean-data:/data/clean
      - ./github-app.pem:/data/clean/github-app.pem:ro
    environment:
      - GITHUB_APP_ID=123456
      - GITHUB_PRIVATE_KEY_PATH=/data/clean/github-app.pem
      - GITHUB_WEBHOOK_SECRET=your-webhook-secret
```

### 4. Install the App

Install the GitHub App on your organization or repositories. Clean will receive an `installation` webhook and begin indexing all selected repositories.

## Webhook events

Clean handles three webhook event types:

| Event | Action | What happens |
|-------|--------|-------------|
| `installation` | `created` | Clones and indexes all selected repos |
| `installation` | `deleted` | Removes all repos and index data |
| `push` | — | Pulls latest changes and re-indexes |
| `ping` | — | Returns `{"status": "pong"}` |

### Webhook security

All webhooks are verified using HMAC-SHA256 signature validation via the `X-Hub-Signature-256` header. If `GITHUB_WEBHOOK_SECRET` is not configured, the webhook endpoint rejects all requests with a `403`.

## Indexing without the GitHub App

You don't need the GitHub App for basic usage. You can index any public repository through the API:

```bash
curl -X POST "http://localhost:8000/repos/index?repo=owner/repo" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Or via MCP:

```
index_repo(repo="owner/repo")
```

The GitHub App adds automatic re-indexing on push and support for private repositories.

<Note>
  Clean enforces license repo limits. If a GitHub App installation includes more repos than your license allows, only the first N repos (up to your limit) will be indexed. Check your license tier for details.
</Note>
