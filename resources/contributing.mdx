## Quick start

```bash
git clone https://github.com/useclean/Clean.git
cd Clean
make setup    # Install deps, generate .env files
make dev      # Start server on localhost:8000
```

No license key or Docker needed for development.

## Make targets

| Command | Description |
|---------|-------------|
| `make dev` | Start server with hot reload |
| `make test` | Run all tests |
| `make test-unit` | Unit tests only (fast) |
| `make lint` | Run linters |
| `make clean` | Tear down Docker + remove .venv |

## Project structure

```
src/clean/
  core/           Config, models, license validation, error types
  parsing/        Tree-sitter parsers (Python, JavaScript, TypeScript)
  indexing/       File scanning, code indexing, incremental reindex
  embedding/      Local embedding model (all-MiniLM-L6-v2)
  search/         Semantic search + context expansion
  storage/        LanceDB vector store
  formatting/     Output formatters (TOON, JSON, Rich)
  transport/      API server (FastAPI) + MCP server (stdio & SSE)
  db/             SQLite metadata store, API key store
  repo/           Git clone/pull manager
  jobs/           Background job worker
  github/         GitHub App client + webhook handler
  stats/          Token savings tracker
  util/           Logging, security, hashing

tests/
  unit/           Fast, isolated tests
  integration/    Tests with real indexing/search
  e2e/            End-to-end MCP server tests
```

## Prerequisites

- **Python 3.10+**
- **Git**
- **GNU Make** — comes with macOS/Linux

### Windows

Run the setup script — it detects missing tools and walks you through installing them:

```powershell
python scripts/dev-setup.py
```

## Running tests

```bash
make test          # All tests
make test-unit     # Unit tests only (fast)
```

Coverage minimum is 60% — CI will fail if you drop below that.

## Making changes

1. Open an issue describing the change
2. Fork the repo and create a branch from `main`
3. Name your branch descriptively: `fix/sse-middleware`, `feat/go-parser`, `docs/api-examples`

### Code style

- Follow existing patterns in the codebase
- Use type hints for function signatures
- Keep functions focused — one function, one job
- No unnecessary abstractions

### Adding a new language parser

Clean uses tree-sitter for parsing. To add a new language:

1. Add the tree-sitter grammar to `pyproject.toml` dependencies
2. Create `src/clean/parsing/yourlang.py` following the pattern in `python.py` or `javascript.py`
3. Register the parser in `src/clean/parsing/registry.py`
4. Add tests in `tests/unit/parsing/`

## Submitting a PR

1. Make sure all tests pass: `make test`
2. Keep commits focused — one logical change per commit
3. Write a clear PR description explaining what and why
4. Link the related issue
