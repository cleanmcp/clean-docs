## Connection

Clean exposes an MCP server over two transports:

| Transport | Endpoint | Use case |
|-----------|----------|----------|
| **SSE** | `GET /mcp/sse` | Remote agents (Claude Code, Cursor) |
| **Stdio** | `clean` CLI command | Local agents, direct integration |

See [Connect your AI agent](/guides/connect-agent) for setup instructions.

## search_code

Semantic code search across an indexed repository. This is the primary tool — agents should use it first for any code question.

### Input schema

```json
{
  "query": "function that validates email format",
  "repo": "owner/repo",
  "branch": "main",
  "cwd": "/Users/dev/project",
  "top_k": 5,
  "depth": 1
}
```

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `query` | string | Yes | — | Natural language description of what you're looking for |
| `repo` | string | No | Auto-detected | Repository in `owner/repo` format |
| `branch` | string | No | Auto-detected | Git branch to search |
| `cwd` | string | No | — | Agent's working directory (for auto-detection) |
| `top_k` | integer | No | `5` | Number of results (1-50) |
| `depth` | integer | No | `1` | Call graph expansion depth (0-3) |

### Auto-detection

When `repo` is omitted, Clean resolves it in order:

1. **Default repo** — if the MCP server was started with `--repo`
2. **Git remote** — parsed from `git remote get-url origin` using the `cwd` parameter
3. **Single repo** — if only one repository is indexed, use it automatically
4. **Error** — if multiple repos are indexed, ask the user to specify

Branch is similarly auto-detected from `git rev-parse --abbrev-ref HEAD` when `cwd` is provided.

### Response

Returns formatted search results with:
- Full source code of matching functions
- File path and line numbers
- Similarity scores
- Call graph context (callers, callees, same-file functions)
- An "Additional matches" list for efficient follow-up

## index_repo

Index a GitHub repository for semantic search.

### Input schema

```json
{
  "repo": "facebook/react",
  "branch": "main",
  "timeout": 600
}
```

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `repo` | string | Yes | — | Repository in `owner/repo` format |
| `branch` | string | No | Default branch | Git branch to index |
| `timeout` | number | No | `600` | Timeout in seconds (max: 3600) |

### Behavior

- Clones the repository, parses all supported files, embeds entities, stores in LanceDB
- If already indexed, returns immediately with entity count
- If already in progress, returns current status
- Detects if the same repo is indexed under a different owner and notifies the agent

## list_repos

List all indexed repositories. No parameters.

### Response

```
Indexed repositories:
  facebook/react                         | ready    | 1,247 entities | javascript | react, jsx
  owner/backend                          | ready    | 342 entities   | python | fastapi, web
```

## get_token_savings

Show cumulative token savings from using compact output format vs raw JSON.

### Input schema

```json
{
  "reset": false
}
```

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `reset` | boolean | No | `false` | Reset all statistics |

### Response

Returns a formatted report showing total searches, character savings, estimated token savings, and average savings percentage.
