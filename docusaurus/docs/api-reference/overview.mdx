## Base URL

```
http://localhost:8000
```

In production, replace with your deployment URL.

## Authentication

All endpoints (except `/` and `/health`) require a Bearer token:

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  http://localhost:8000/search?q=login&repo=owner/repo
```

Clean supports two types of API keys:

| Key type | Format | Set via |
|----------|--------|---------|
| **Admin key** | Any string | `CLEAN_API_KEY` env var |
| **Dashboard key** | `clean_sk_*` | Synced from dashboard |

Admin keys have full access. Dashboard keys can be scoped to specific repositories and permissions (`search`, `index`, `admin`).

In dev builds (`CLEAN_BUILD_MODE=dev`), authentication is disabled entirely.

## Rate limiting

Requests are rate-limited per client IP using a sliding window:

| Limit | Default |
|-------|---------|
| Per IP | 60 requests/minute |
| Per API key | 120 requests/minute |

Rate-limited requests receive:

```
HTTP 429 Too Many Requests
Retry-After: 60
```

## Response format

### Search results

The `/search` endpoint returns TOON format (compact tabular output) with token savings metadata:

```json
{
  "results": "results\n  name | file | line | sim\n  login | src/auth.py | 42 | 95%\n...",
  "result_count": 5,
  "token_savings": {
    "chars_saved": 1247,
    "percent_saved": 34.92
  }
}
```

### Error responses

Errors follow a consistent format:

```json
{
  "detail": "Repository not found: owner/repo"
}
```

| Status | Meaning |
|--------|---------|
| `400` | Invalid request (bad repo format, invalid branch) |
| `401` | Missing `Authorization` header |
| `402` | License repo limit exceeded |
| `403` | Invalid API key or insufficient permissions |
| `404` | Repository not indexed |
| `409` | Repository not ready (still indexing) |
| `429` | Rate limited |

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | Health check (lightweight) |
| `GET` | `/health` | Deep health check (embedding model + LanceDB) |
| `GET` | `/search` | Search indexed code |
| `POST` | `/repos/index` | Start indexing a repository |
| `GET` | `/repos` | List all indexed repositories |
| `GET` | `/repos/{owner}/{repo}/status` | Get indexing status |
| `POST` | `/repos/{owner}/{repo}/cancel` | Cancel an indexing job |
| `DELETE` | `/repos/{owner}/{repo}` | Delete a repository from the index |
| `GET` | `/stats` | Get token savings statistics |
| `POST` | `/github/webhook` | GitHub App webhook receiver |

## MCP endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/mcp/sse` | SSE stream for MCP clients |
| `POST` | `/mcp/messages/` | MCP tool call submissions |

Both MCP endpoints require Bearer token authentication.
