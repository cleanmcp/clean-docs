## Endpoint

```
POST /repos/index
```

## Authentication

Requires Bearer token with `index` scope.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `repo` | string | Yes | — | Repository in `owner/repo` format |
| `branch` | string | No | Default branch | Git branch to index |

## Example request

```bash
curl -X POST "http://localhost:8000/repos/index?repo=facebook/react" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Index a specific branch:

```bash
curl -X POST "http://localhost:8000/repos/index?repo=owner/repo&branch=develop" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Response

Returns immediately — indexing runs in the background.

### Queued (new repo)

```json
{
  "status": "queued",
  "repo": "facebook/react",
  "message": "Indexing started. Check /repos/{owner}/{repo}/status for progress."
}
```

### Already indexed

```json
{
  "status": "already_indexed",
  "repo": "facebook/react",
  "branch": null,
  "entity_count": 1247
}
```

### In progress

```json
{
  "status": "in_progress",
  "repo": "facebook/react",
  "branch": null,
  "current_status": "indexing"
}
```

## Indexing pipeline

When a repo is queued, Clean runs through these phases:

1. **Cloning** — `git clone` the repository
2. **Scanning** — walk the file tree, filter by supported extensions
3. **Parsing** — extract functions, classes, and methods with tree-sitter
4. **Embedding** — batch-embed entities with `all-MiniLM-L6-v2`
5. **Storing** — upsert into LanceDB with deduplication
6. **Call graph** — compute caller/callee relationships

Check progress with [`GET /repos/{owner}/{repo}/status`](/api-reference/repo-status).

## Error responses

| Status | Condition |
|--------|-----------|
| `400` | Invalid repo format or branch name |
| `402` | License repo limit exceeded |
| `403` | API key doesn't have access to this repository |
