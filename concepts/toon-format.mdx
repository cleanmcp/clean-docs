## What is TOON?

TOON (Token-Optimized Object Notation) is Clean's compact output format for search results. It uses a tabular layout that conveys the same information as JSON while using **30-40% fewer tokens**.

This matters because every token your AI agent consumes costs money. At scale, TOON saves real dollars on every query.

## Token savings tracking

Clean tracks savings on every search. Both the MCP server and the HTTP API measure the character difference between JSON and TOON output, then estimate token savings (1 token ≈ 4 characters).

Stats are persisted to `~/.clean/stats.json` and accumulate across sessions:

```
=======================================================
           Clean Token Savings Statistics
=======================================================

  Total Searches:           142

  Character Savings:
    JSON output (total):    1,247,832 chars
    TOON output (total):    812,091 chars
    Characters saved:       435,741 chars

  Estimated Token Savings:
    JSON tokens (est):      311,958
    TOON tokens (est):      203,023
    Tokens saved (est):     108,935

  Average Savings:          34.92%

  This Session:
    Searches:               12
    Tokens saved:           8,421
=======================================================
```

The MCP server exposes a `get_token_savings` tool so your agent can check savings during a session.

## How TOON works

TOON is a line-oriented tabular format:

- **Section headers** are plain words (`results`, `context`)
- **Tables** use `|` separators with consistent column alignment
- **Nested data** uses indentation (2 spaces per level)
- **No braces, brackets, or quotes** — minimal syntax overhead
- **Numbers and percentages** are human-readable (`95%` not `0.9503`)

The format is configurable via `ToonFormatterConfig`:

| Option | Default | Description |
|--------|---------|-------------|
| `columns` | `["name", "file", "line", "sim"]` | Columns in the results table |
| `row_indent` | `2` | Spaces before each row |
| `column_separator` | `" \| "` | Delimiter between columns |
| `max_code_lines` | `20` | Max lines of code to include |

## Output formats

Clean supports three output formats:

| Format | Used by | Optimized for |
|--------|---------|--------------|
| **TOON** | MCP server, HTTP API | AI agents (minimal tokens) |
| **JSON** | HTTP API (for comparison) | Programmatic access |
| **Rich** | CLI / debugging | Human readability |

The HTTP `/search` endpoint returns both TOON and JSON in the response, along with a `chars_saved` field showing the savings for that specific query.
